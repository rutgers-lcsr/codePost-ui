## 5) Issue Tracking and Support via GitHub

To track bugs, enhancements, and tasks, use GitHub Issues in the UI repository:

- Issues URL: <https://github.com/rutgers-lcsr/codePost-ui/issues>

When opening an issue, include:

1. **Title**: short and actionable
2. **Type**: bug, enhancement, documentation, or question
3. **Environment**: OS, browser, Node version, deployment mode
4. **Steps to reproduce**
5. **Expected behavior**
6. **Actual behavior**
7. **Logs/screenshots** (if available)
8. **Impact/priority**

### Suggested issue title format

`[area] concise description`

Examples:

- `[grading] rubric comments not saving in batch edit`
- `[deploy] UI fails to load when REACT_APP_API_URL is unset`
